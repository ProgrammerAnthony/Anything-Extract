# 深模块（Deep Modules）

摘自 “A Philosophy of Software Design”：

**深模块（Deep module）** = 小接口 + 大量实现细节

```
┌─────────────────────┐
│   小接口（Small Interface）   │  ← 很少的方法，参数简单
├─────────────────────┤
│                     │
│                     │
│  深实现（Deep Implementation）  ← 复杂逻辑被隐藏起来
│                     │
│                     │
└─────────────────────┘
```

**浅模块（Shallow module）** = 大接口 + 很少实现（应避免）

```
┌─────────────────────────────────┐
│       大接口（Large Interface）           │  ← 方法很多，参数复杂
├─────────────────────────────────┤
│  很薄的实现（Thin Implementation）            │  ← 只是简单透传
└─────────────────────────────────┘
```

在设计接口时，反问：

- 我能减少方法数量吗？
- 我能简化参数吗？
- 我能把更多复杂度隐藏到实现内部吗？
